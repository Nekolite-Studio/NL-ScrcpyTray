# NL-ScrcpyTray 開発ロードマップ

## 1. プロジェクトの目的

`NL-ScrcpyTray` は、高機能なAndroid画面ミラーリングツール `scrcpy` を、Windowsユーザーがより直感的かつ手軽に利用できる環境を提供することを目指します。
「ただ動く」ツールから、誰もがストレスなく使える「洗練された」プロダクトへと進化させていきます。

## 2. バージョンごとの開発計画

### Version 1.0: 基盤構築 (Foundation)

**目標:** アプリケーションを「ポータブルな単体ツール」として完成させ、基本的な利便性を確立する。

-   **[済] プロトタイプ完成:**
    -   タスクトレイからの手動起動・停止。
    -   基本的な `scrcpy` オプションのハードコードによる制御。

-   **[完了] 設定の永続化:**
    -   `settings.json` を導入し、ユーザーが行った設定（画質、自動起動など）を次回起動時にも維持できるようにする。

-   **[完了] `scrcpy` の内蔵:**
    -   `scrcpy` 本体をアプリケーションフォルダに同梱し、ユーザーが別途インストールやパス設定を行う手間をなくす。
    -   コード内のパス指定を絶対パスから相対パスに変更し、どこに置いても動作するポータブル性を確保する。

-   **[完了] 基本的な通知機能:**
    -   USBデバイスの接続・切断を検知し、Windowsの通知（バルーンチップ）でユーザーにフィードバックする。

### Version 1.x: ユーザー体験の向上 (UX Enhancement)

**目標:** より多くのユーザーが快適に使えるように、操作性を向上させ、便利な機能を追加する。

-   **専用設定ウィンドウの実装:**
    -   コンテキストメニューの「設定...」から専用のUIを開き、ビットレートや解像度などの詳細な `scrcpy` パラメータをGUIで直感的に調整できるようにする。

-   **ワイヤレス接続 (TCP/IP) のサポート:**
    -   USB接続中のデバイスに対して `adb tcpip 5555` コマンドを自動で発行し、Wi-Fi経由でのミラーリングにシームレスに切り替える機能を追加する。

-   **マルチデバイス対応:**
    -   複数のAndroidデバイスが接続された場合に、どのデバイスをミラーリング対象にするかを選択できるUIを提供する。

-   **ブランド力の強化:**
    -   アプリケーションのアイコンをオリジナルデザインに変更する。
    -   デバイスの状態（接続中、切断、エラー）に応じてトレイアイコンの色や形が変化するようにし、視認性を高める。

### Version 2.0: 高度な機能と拡張性 (Power Features & Extensibility)

**目標:** `scrcpy` のポテンシャルを最大限に引き出し、開発者やパワーユーザー向けの高度な機能を提供する。

-   **画面録画機能の統合:**
    -   ミラーリング中の画面を、ボタン一つで `.mp4` ファイルとしてPC上に録画できる機能を追加する。

-   **クリップボード共有の制御:**
    -   PCとAndroidデバイス間のクリップボード同期を、UI上から任意にON/OFFできるスイッチを設ける。

-   **スタートアップ登録機能:**
    -   アプリケーションの設定画面から「Windows起動時に実行する」を選択できるようにし、OSのスタートアップに自動で登録する機能を追加する。

-   **プラグイン/スクリプト機能:**
    -   特定のイベント（例: 特定のアプリが起動した時）に、ユーザーが定義したカスタムスクリプトを実行できるような拡張性を持たせる。

このロードマップは、プロジェクトの進捗やユーザーからのフィードバックに応じて、随時見直されるものとします。

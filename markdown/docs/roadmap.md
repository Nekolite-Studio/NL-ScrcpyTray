# 💡 NL-ScrcpyTray 開発ロードマップ

## 1. プロジェクトの目的

`NL-ScrcpyTray` は、高機能な **Android画面ミラーリングツール `scrcpy`** を、Windowsユーザーがより直感的かつ手軽に利用できる環境を提供することを目指します。
「ただ動く」ツールから、誰もがストレスなく使える **「洗練された」プロダクト** へと進化させていきます。

---

## 2. バージョンごとの開発計画

## Version 1.0: 基盤構築 (Foundation) [完了]

**目標:** アプリケーションを「**ポータブルな単体ツール**」として完成させ、基本的な利便性を確立する。

* **[完了] プロトタイプ完成:**
    * タスクトレイからの手動起動・停止。
    * 基本的な `scrcpy` オプションのハードコードによる制御。
* **[完了] 設定の永続化:**
    * `settings.json` を導入し、ユーザーが行った設定（画質、自動起動など）を次回起動時にも維持できるようにする。
* **[完了] `scrcpy` の内蔵:**
    * `scrcpy` 本体をアプリケーションフォルダに同梱し、ポータブル性を確保。
* **[完了] 基本的な通知機能:**
    * **USBデバイスの接続・切断** を検知し、Windowsの通知（バルーンチップ）でユーザーにフィードバックする。

---

## Version 1.x: ユーザー体験の向上と高度なデバイス管理 (UX Enhancement) [完了]

**目標:** **WPF + WebView2(React)アーキテクチャ**への刷新による最新のUI/UXと、複数デバイスの同時利用を可能にする堅牢な管理機能を提供する。

* **[完了] UI/UXの全面刷新:**
    * **WPF + WebView2 (React) アーキテクチャ** を導入。
    * `mockups/setting-ui/1.html` に準拠した、モダンでインタラクティブなGUIを実装。
    * フロントエンドをコンポーネント指向で再構築し、Viteによるビルドプロセスを確立。
* **[完了] 高度なマルチデバイス管理:**
    * **複数デバイスの同時ミラーリング**に対応。
    * ドラッグ＆ドロップによるデバイス優先順位の並べ替え。
    * デバイスごとの詳細なプロファイル管理（有線/無線別設定）。
* **[完了] 有線・無線のスマートハンドオーバー:**
    * Wi-Fi接続中にUSBが接続された場合、**自動で有線接続へ切り替える**機能。
    * (USB→Wi-Fiの自動移行は、安定性の観点から次期バージョンでの検討課題とする)
* **[完了] 詳細な画質・録画設定:**
    * ビットレート、FPS、コーデック、バッファサイズなどの詳細な手動調整UI。
    * 画面・音声の録画機能と保存先設定。
* **[完了] タスクトレイ常駐機能の再実装:**
    * 新UIアーキテクチャへの移行に伴い、ウィンドウを閉じてもバックグラウンドで動作し続けるタスクトレイ常駐機能を再実装。

---

## Version 2.0: 拡張性とエコシステム (Power Features & Extensibility)

**目標:** `scrcpy` のポテンシャルを最大限に引き出し、**開発者やパワーユーザー向け**の高度な機能を提供する。

* **クリップボード共有の制御:**
    * PCとAndroidデバイス間のクリップボード同期を、UI上から任意に **ON/OFF** できるスイッチを設ける。
* **スタートアップ登録機能:**
    * アプリケーションの設定画面から「**Windows起動時に実行する**」を選択できるようにし、OSのスタートアップに自動で登録する機能を追加する。
* **プラグイン/スクリプト機能:**
    * 特定のイベント（例: 特定のアプリが起動した時）に、ユーザーが定義した**カスタムスクリプトを実行**できるような拡張性を持たせる。
